APP LAYOUT:

Index file for every module.
Every page has - Page header (title, description), Page body, Page actions
Page body - has Form and Grid

CRUD - Read - Get all items (List - Can be used for populating a table or a dropdown with items), Get single item (Details)

---

List page:

- Create reusable component from UI library Data table. This ensures consistency for all modules.
- List page supports - Search, filter, sort, pagination, inline actions - edit & delete (using API parameters).
- Clicking on entity name takes us to details page of that entity.
- Datatable has Add button in upper toolbar to take us to add page.
- Add, edit, delete, details possible via list page.

---

Add/Edit page:

- Same component for add and edit, depends on the mode.
- For edit - entity id should be in url. Eg: /user/327812/edit
- Fetch the details from details API and fill the form fields.
- Edit page has two page actions - 1.Cancel, 2.Update

---

Details page:

- Fetch the details from details API and fill the form fields.
- Form fields are readonly and disabled.
- For details - entity id should be in url. Eg: /user/327812
- Details page has two page actions - 1.Back, 2.Edit

Dropdown usecases:

Case 1 - Single Select + Detail Card:

The user selects an option from a single-select dropdown. Based on the selected value, a card or tile is displayed below the dropdown showing the details of the selected entity.

Case 2 - Multi Select + Table with Remove:

The user selects multiple options from a multi-select dropdown. The selected items are displayed in a table below the dropdown, with each item shown as a separate row. Each row contains a remove (x) icon that allows the user to delete the corresponding item from both the table and the dropdown selection.

Case 3 - Add Button → Form → Append:

An “Add” button is provided alongside the single-select or multi-select dropdown. Clicking the button opens an input form, which may appear as a modal dialog, side panel, or separate page. Upon successful submission, the newly created entity is added to the dropdown options and automatically reflected in the associated card or table view.

Case 4 - Paginated Dropdown (Load More):

The dropdown options are populated through a list API that may contain thousands of records. Instead of fetching all items at once, the dropdown supports incremental data loading using pagination or a “Load More” mechanism to improve performance and scalability.

Case 5 - Search in Dropdown:

The dropdown provides a search capability that allows users to quickly filter and locate options by entering keywords or partial text.

Case 6 - Async Search Dropdown (Typeahead / Autocomplete):

Instead of preloading all options, the dropdown fetches matching results dynamically as the user types. API calls are triggered using a debounced mechanism to minimize unnecessary requests and improve responsiveness.

Case 7 - Cascading / Dependent Dropdowns:

The available options in one dropdown depend on the selection made in another dropdown. Changing the parent selection dynamically updates the child dropdown options accordingly.

Principles

- If a column in list page has multiple values, instead of comma seperated, show a view link and open modal.
- Three types of pages - 1.Route page/ main page, 2.Side panel, 3.Modal pop up
- To add entity, open side panel. From side panel, open a modal.
- Side drawer - should be visible below app bar.
- Stationary/Sticky header - Content is very long so scrollbar and footer at end. Scrollable page - occupies full height.
- Maintain the routes in a seperate file - as an array of objects. Each object has - path, component.
- Colour constants and string constants for entire project should be in one seperate file
- Setup an auth provider and context - Routes must be guarded using private routes.
- On clicking a list item, user is redirected to details page - Item ID should be visible in the url as a path parameter.
- Add and Edit page should be the same page component not different. There should be a pagemode such as ADD or EDIT based on which operations should be performed.
