WORK FLOW UNDERSTANDING:

Usecase:

Entities/Modules:

1. Student => Student signup, Student onboard, interview
2. Institute => Institute onboard
3. Institute co-ordinator => IC onboard
4. Admin => Onboarding

* All users will be registered in Cognito. In cognito there will be 3 groups/roles – admin, icadmin, students.

Onboarding Sequence:

1. Bluestar admin registers themselves through a link, clicking it opens registration form and upon submit gets email verification link. On success, admin added in cognito with role admin.
2. Admin does "Add Institute" by fillling form and adding basic details and email of institute coordinator. Then configure number of invitiations, selections.
3. Admin sends invite to IC for onboarding himself and onboarding institute.
4. When IC onboards, he/she gets registered on Cognito with role icadmin. Then he completes institute details and onboards institute.
5. IC checks students from his institute which match the criteria for scholarship and onboard student. IC adds Student emails upto specific number and sends invitations.
6. Student registers through onboarding email and registers on cognito with role student.
7. Student interview, scheduling, selection process, if passed then processing fee and giving invoice.

FORM:

Formik, Form (form state management), Yup (validation), Material-UI (UI components), and Emotion (CSS-in-JS styling).

1. Form State Management (Formik)

- Formik handles form state, validation, and submission
- Key props:
  - `initialValues` - Object with all form field values
  - `validationSchema` - Yup schema for validation
  - `onSubmit` - Callback function when form is submitted
- Render props pattern: `{({ values, errors, touched, setFieldValue }) => (...)}`
  - `values` - Current form data
  - `errors` - Validation errors
  - `touched` - Fields that have been interacted with
  - `setFieldValue` - Function to update specific field values

2. Validation with Yup

```javascript
Yup.object({
  firstName: Yup.string().trim().required('First name is required'),
  email: Yup.string().email('Invalid email').required('Email is required'),
  phone: Yup.string().matches(/^[0-9]{10}$/, 'Phone number must be 10 digits'),
  password: Yup.string().min(6).required('Password is required'),
  confirmPassword: Yup.string()
    .oneOf([Yup.ref('password')], 'Passwords must match'),
  acceptTerms: Yup.boolean().oneOf([true], 'You must accept the terms'),
})
```

- Key validators: `.required()`, `.email()`, `.min()`, `.matches()` (regex), `.oneOf()`
- Cross-field validation: `Yup.ref('password')` references another field

3. Form Fields & Components

| Toggle Switch   | `<Switch>`                               | For newsletter subscription (optional)     |

4. Error Display Pattern

```javascript
{touched.firstName && errors.firstName && (
  <Typography variant="caption" color="error">
    {errors.firstName}
  </Typography>
)}
```

- Errors only show after field is touched (user interaction)
- Prevents overwhelming users with immediate errors
- Uses MUI `<Typography>` for consistent styling

FormikErrorFocus - Quick Notes

What: Auto-scrolls & auto-focuses first form error
Import: `import FormikErrorFocus from 'formik-error-focus';`
Usage: Place `<FormikErrorFocus />` at end of `<Form>` (inside Formik)
Why: Better UX - users don't hunt for errors on long forms
Props: None needed - reads from Formik context automatically
Benefit: Improves accessibility for keyboard + screen reader users
